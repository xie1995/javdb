# 数据分析与报告

数据分析与报告同时关联仪表盘中的 `报告` 与 `设置 → 报告`。

当前版本里，这两者的分工非常明确：

- `报告`：生成、查看、导出和管理历史月报
- `设置 → 报告`：配置统计范围、聚合参数和 AI 提示词输入相关参数

## 报告页现在是什么样

当前报告页的真实结构重点包括：

- 时间范围选择
- 当前人设显示
- 生成报告
- 历史月报列表
- 预览区域
- 导出
- 删除所选
- 刷新历史

这说明当前报告系统已经不是“简单看几个统计数字”，而是一个完整的月报生成与查看流程。

## 当前最适合怎么用

### 场景 1：生成某个时间段的报告

1. 打开仪表盘 `报告`
2. 选择起止月份
3. 点击 `生成报告`
4. 在右侧预览区查看结果

这是最标准的主流程。

### 场景 2：管理历史月报

报告页左侧会维护历史月报列表。

你可以在这里：

- 查看历史报告
- 导出
- 刷新历史
- 删除所选

当前文档更建议把这部分理解成“报告档案区”。

### 场景 3：先调设置，再生成报告

如果你觉得报告口径不对、排行不合理、统计范围不符合预期，不应只在报告页反复重试，而应去：

- `设置 → 报告`

先调整配置，再重新生成。

## 设置 → 报告 负责什么

根据当前真实设置页，这里主要负责：

- 数据源模式
- 统计状态范围
- Top N 排行数量
- 其他聚合参数
- 与 AI 提示词输入相关的本地统计配置

它影响的是“报告怎么统计、怎么组织输入”，而不是替代报告页本身。

## 当前更稳妥的理解方式

不建议继续沿用旧文档里“设置面板首页看统计、点击生成月报按钮”的简单想象。

当前应该这样理解：

- 报告页是结果生成和阅读入口
- 报告设置页是参数配置入口
- 两者一起构成完整的报告系统

## 报告里通常适合看什么

虽然具体内容会受设置和数据影响，但你可以预期它更适合回答这类问题：

- 某段时间看了多少内容
- 哪些标签、类型或偏好更突出
- 观看与浏览之间有什么差异
- 历史报告之间有什么变化趋势

## 报告和数据管理的关系

报告不是独立生成“新数据”，而是对已有记录做再组织和再解释。

所以如果原始记录本身不完整，报告也会受影响。

建议顺序通常是：

1. 先把 [视频标记](/guide/video-marking) 和 [数据管理](/guide/data-management) 用稳定
2. 再来使用报告页
3. 如果统计口径不合适，再去 [报告设置](/guide/settings-report)

## 什么时候值得关注报告设置

你遇到下面这些情况时，优先去看设置页：

- 想把“仅已看”改成“已看 + 已浏览”一起统计
- 想调整 Top N 排行规模
- 想改变聚合口径
- 想让本地统计与 AI 输入更匹配
