# 115 网盘集成

115 网盘集成同时关联仪表盘中的 `115任务` 与 `设置 → 115网盘`。

当前版本的 115 功能已经不是早期的“只依赖网页登录态直接推送”模式，而是以 **115 v2 离线下载** 为主，核心包括：

- 在 `设置 → 115网盘` 中启用 115 离线下载
- 通过 **OpenList 手动获取**、**OpenList 扫码** 或 **自有应用扫码** 完成授权
- 保存并验证 `refresh_token` / `access_token`
- 为详情页磁力链接执行 `推送115`
- 在 `115任务` 页面查看、搜索、刷新、删除或清空任务
- 使用默认目录 ID、验证次数、最大失败数、自动通知等参数控制离线任务行为

## 先区分两个入口

### 设置 → 115网盘

这里负责 **开关、授权、token、目录、验证和测试**。

你会在这里看到这些真实配置项：

- 启用 115 离线下载
- 获取方式
  - 借用 OpenList 手动获取
  - 借用 OpenList 扫码
  - 自有应用扫码
- 接口域名（v2）
- `refresh_token` / `access_token`
- token 有效性验证与状态展示
- 默认目录 ID
- 验证次数
- 最大失败数
- 自动通知
- 测试搜索
- 115 日志

### 115任务

这里负责 **任务查看和管理**。

当前任务页支持的实际能力包括：

- 手动添加下载任务
- 搜索任务名称
- 刷新任务列表
- 清空所有任务
- 查看统计信息
- 分页浏览任务

## 这页适合谁

如果你现在想做的是下面这些事，这页就是当前版本对应的正确文档：

- 配置 115 v2 授权，而不是单纯依赖网页登录态
- 给详情页磁力按钮接入 115 离线下载
- 指定离线任务默认目录 ID
- 查看任务是否已提交、失败还是需要重试
- 配合“推送成功后自动标记已看”使用详情页推送按钮

## 功能现状总览

### 1. 详情页推送到 115

在影片详情页的磁力区域，扩展会为磁力链接提供 `推送115` 按钮。

推送成功后会：

- 显示成功提示
- 记录 115 推送日志
- 在开启相关选项时，自动把影片标记为“已看”

### 2. 批量下载能力

代码里仍存在批量下载逻辑，但你应以当前界面实际暴露的入口为准。

对于普通使用者，当前最稳定、最明确的主流程仍然是：

- 在详情页逐条磁力执行 `推送115`
- 或在 `115任务` 页面直接手动添加任务链接

如果后续界面重新暴露批量入口，再单独补充对应文档更稳妥。

### 3. 自动标记“已看”

115 推送成功后自动标记“已看”这个能力仍然存在，但它不在 `设置 → 115网盘` 中配置，而是在视频增强相关设置里控制。

也就是说：

- `设置 → 115网盘` 负责 115 授权、目录和任务行为
- 自动标记“已看”属于详情页视频增强联动能力

如果你只想把任务推到 115，不想同步改观看状态，就需要去对应的视频增强设置里关闭它。

## 首次配置流程

### 第一步：启用 115 离线下载

1. 打开仪表盘
2. 进入 `设置`
3. 打开 `115网盘`
4. 开启 `启用 115 离线下载`

如果未启用，详情页 `推送115` 按钮会直接提示 115 功能未开启。

### 第二步：选择授权方式

当前版本支持三种方式。

### 方式 A：借用 OpenList 手动获取

适合：

- 没有自有 115 开放平台应用
- 想快速完成授权
- 能接受手动复制 token

基本流程：

1. 在 `获取方式` 里选择 `借用 OpenList 手动获取`
2. 点击设置页里的获取地址
3. 在目标页面选择 `115 网盘 (OAuth2)`
4. 按提示登录并获取 `refresh_token` / `access_token`
5. 粘贴回设置页下方的 token 区域
6. 点击验证有效性

### 方式 B：借用 OpenList 扫码

适合：

- 不想手动复制 token
- 没有自有 115 应用
- 可以使用 115 客户端扫码

基本流程：

1. 在 `获取方式` 中选择 `借用 OpenList 扫码`
2. 点击生成二维码
3. 用 115 手机客户端扫码并确认
4. 等待扩展自动保存新的 token
5. 查看授权状态是否成功

### 方式 C：自有应用扫码

适合：

- 你自己有 115 开放平台 APP ID
- 想使用自己的应用完成 PKCE 扫码授权

基本流程：

1. 在 `获取方式` 中选择 `自有应用扫码`
2. 输入 APP ID
3. 点击生成二维码
4. 使用 115 手机客户端扫码确认
5. 等待扩展写入新的 token

### 第三步：检查 token 状态

当前版本会显式显示 token 状态，而不是像旧版那样只靠“能不能推送成功”来判断。

你需要重点确认：

- `refresh_token` 是否有效
- `access_token` 是否有效
- 是否能通过“验证有效性”检查
- 是否出现过期、失效、限流等状态提示

如果 token 状态异常，优先重新授权，而不是直接怀疑磁力链接有问题。

### 第四步：配置默认目录与任务参数

### 默认目录 ID

现在不是填写“目录路径字符串”，而是配置 **默认目录 ID**。

这意味着：

- 你需要使用 115 侧真实目录对应的 ID
- 扩展推送离线任务时会把任务提交到这个目录 ID
- 如果目录 ID 错误，任务可能提交失败或落到错误目录

### 验证次数

用于下载后验证文件的重试次数。

建议：

- 日常使用保留默认值或 3–5 次
- 不要盲目调得太高，否则失败时等待会更久

### 最大失败数

用于批量/连续任务场景下控制允许的失败上限。

- `0` 表示不限制
- 普通用户保持默认即可

### 自动通知

开启后，任务完成时会自动显示通知。

适合希望及时知道离线任务结果的用户。

## 日常使用流程

### 场景 1：从详情页推送磁力到 115

1. 打开影片详情页
2. 找到磁力列表
3. 点击某条磁力旁边的 `推送115`
4. 等待返回结果

成功后通常会看到：

- 按钮状态变化
- 成功提示
- 相关日志记录

如果你同时开启了“115 推送成功后自动标记已看”，还会继续触发观看状态联动。

### 场景 2：在 115任务 页面手动加任务

1. 打开仪表盘顶部 `115任务`
2. 在输入框里粘贴下载链接
3. 点击 `添加任务`
4. 再通过刷新、搜索和分页查看结果

当前任务输入框支持：

- HTTP(S)
- FTP
- 磁力链
- 电驴链接

### 场景 3：在 115任务 页面排查问题

当前任务页可以直接用于运维排查：

- 看统计信息是否异常
- 搜索某个任务是否已经入列
- 刷新任务列表确认最新状态
- 删除单个异常任务或清空全部任务

## 配额、搜索与日志

### 配额信息

当前版本包含 115 配额相关能力，并会缓存部分配额数据。

你可以把它理解成：

- 扩展会尝试读取 115 离线下载/空间配额信息
- 配额异常时，某些任务提交可能失败
- 如果界面提示空间不足或接口异常，应优先检查 115 账号侧状态

### 测试搜索

`设置 → 115网盘` 中自带 `测试搜索`。

适合用来确认：

- 当前 token 是否可用
- 115 v2 搜索接口是否能正常返回结果
- 当前 115 授权链路是否已经打通

### 115 日志

设置页还提供 115 日志区，可以：

- 刷新日志
- 清空日志
- 导出日志

如果你遇到授权失败、任务失败、token 刷新失败，这里通常比旧版“只看按钮提示”更有用。

## 常见问题

### 为什么我已经登录 115，还是不能推送？

因为当前版本主要依赖 **115 v2 token 授权链路**，不再等同于“浏览器里登录了 115 就一定能用”。

优先检查：

- 是否启用了 `115 离线下载`
- `refresh_token` / `access_token` 是否有效
- 是否完成了扫码或手动获取流程
- 接口域名是否正确

### 为什么文档里以前说填目录路径，现在却是目录 ID？

因为当前版本的实际设置项已经改成 **默认目录 ID**，旧文档仍保留了早期思路，所以才会对不上。

### 为什么任务提交成功了，却没有自动标记“已看”？

因为自动标记开关不在 `设置 → 115网盘`，而是在视频增强相关设置中。

另外，只有推送真正成功后才会进入自动标记逻辑。

### 为什么我在 115 设置页里看不到旧版那种网页登录说明？

因为现在主流程已经围绕 v2 token、扫码授权、OpenList 辅助获取和任务验证展开，旧版说明已经不适合作为主文档。
