# AI 翻译

AI 翻译对应仪表盘中的 `设置 → AI设置`，用于配置模型、提供商、测试调用和对话参数。

它和详情页的翻译入口是配套关系：

- `AI设置` 负责配置
- 页面增强和详情页负责实际触发翻译

## 当前设置页里有什么

根据当前真实设置页，AI设置主要包括：

- 提供商与模型配置
- API 地址 / 密钥类配置
- 发送测试消息
- 对话参数
  - 创造性
  - 最大回复长度
  - 超时时间
  - 系统提示词
  - 是否启用流式输出
- 设置导入导出与重置

## 当前更适合怎么理解它

不建议把它只理解成“标题翻译按钮设置页”。

更准确的理解是：

- 这是扩展里与 AI 能力相关的总配置入口
- 翻译只是最直接的一种使用方式
- 页面中的翻译表现会受到这里的提供商、模型和参数影响

## 常见使用场景

### 场景 1：理解标题和内容

最常见的用法是帮助阅读：

- 标题
- 简介
- 标签
- 页面中的部分文本内容

### 场景 2：验证 AI 配置是否可用

当前设置页已经提供测试消息能力。

所以比起“先到页面里猜为什么没反应”，更推荐：

1. 先在 `AI设置` 中测试调用
2. 确认模型和密钥正常
3. 再回页面看翻译入口是否工作

### 场景 3：调节回答风格和稳定性

如果你觉得结果不理想，可以从这些方向调整：

- 模型选择
- 创造性参数
- 最大回复长度
- 超时时间
- 系统提示词

## 当前推荐配置思路

### 第一步：先让调用成功

先保证：

- 提供商正确
- API 配置正确
- 测试消息可以返回结果

### 第二步：再去看页面效果

调用成功后，再到详情页或相关入口看翻译体验是否符合预期。

### 第三步：最后才调参数

如果基础调用都还不稳定，就不要先急着微调复杂参数。
