# Emby 集成

Emby 集成对应仪表盘中的 `设置 → Emby增强`，用于把站点内容与 Emby 媒体服务器的使用流程连接起来。

## 当前更适合怎么理解它

当前版本里，Emby 集成更像一套“媒体库联动增强”，而不是单独的站外管理器。

你可以把它理解成：

- 在设置页中配置 Emby 连接与相关能力
- 在页面中使用联动入口、跳转或增强结果
- 在需要时配合本地数据管理和页面增强一起使用

## 设置页负责什么

当前应从 `设置 → Emby增强` 进入配置。

你通常会在这里处理：

- Emby 服务器连接信息
- 相关联动配置
- 是否启用 Emby 相关增强能力
- 与页面跳转或匹配行为有关的设置

## 实际使用场景

### 场景 1：从站点跳到 Emby

如果一部作品已经能和 Emby 中的内容形成关联，你通常会希望：

- 更快找到对应媒体项
- 直接跳到 Emby 播放或查看页面

### 场景 2：从 Emby 反向回到站点

如果你的使用习惯是先在媒体库里看，再回到站点补充信息，这类联动也会更有价值。

### 场景 3：把 Emby 当作浏览增强的一部分

当前最稳妥的理解不是“完全依赖 Emby 管理一切”，而是把它看成：

- 页面增强
- 信息联动
- 媒体库跳转

的一部分。

## 当前文档为什么不再写很多固定按钮位置

旧文档里大量写了：

- 某个测试连接按钮
- 某个右侧快捷按钮
- 某个页面一定会出现的跳转按钮

这类写法在功能快速演进时最容易过时。

当前更适合的文档方式是：

- 明确入口在 `设置 → Emby增强`
- 明确能力目标是“匹配、跳转、联动、增强”
- 再结合真实页面判断当前具体展示方式

## 什么时候应该先看这页

如果你符合下面任一情况，就建议先配置 Emby：

- 你已经有在用 Emby 媒体库
- 你希望在站点和媒体库之间减少来回切换成本
- 你希望把页面浏览和媒体播放流程连接起来
